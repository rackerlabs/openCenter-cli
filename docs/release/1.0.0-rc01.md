# openCenter CLI v1.0.0-rc01 — Release Notes

**First Release Candidate** | May 2026

---

## Overview

This is the first release candidate for openCenter CLI 1.0.0 — a command-line tool that transforms a single declarative YAML configuration into a production-ready Kubernetes cluster with GitOps management.

openCenter standardizes cluster deployment across multiple infrastructure providers, providing configuration validation, secrets management, and automated GitOps repository generation.

---

## Core Features

### Cluster Lifecycle Management
- **Initialize** clusters with sensible defaults per provider
- **Validate** configurations with schema, business rules, and provider-specific checks
- **Generate** complete GitOps repositories ready for FluxCD reconciliation
- **Deploy** infrastructure and bootstrap Kubernetes clusters
- **Destroy** clusters with safety confirmations and resource cleanup

### Multi-Provider Support
- **OpenStack** — Full integration with Keystone auth, Nova compute, Neutron networking
- **VMware/vSphere** — Pre-provisioned VM deployment with CSI storage
- **Baremetal** — Kubespray-based provisioning with Ansible
- **Kind** — Local development clusters with optional CNI management

### GitOps Repository Generation
- FluxCD Kustomization manifests with proper dependency ordering
- Base + overlay structure for cluster-specific customization
- Infrastructure layer (Terraform/OpenTofu) and application layer (Helm/Kustomize)
- Automated GitRepository source configuration

### Secrets Management (SOPS + Age)
- Age key pair generation and secure storage
- Encrypt/decrypt secrets in YAML manifests
- Key rotation with re-encryption
- Drift detection between config and deployed secrets
- Expiration checking and backup utilities
- OpenStack Barbican integration for key storage

### Platform Services
20+ pre-configured services available out of the box:
- **Monitoring:** kube-prometheus-stack, Grafana, Alertmanager
- **Logging:** Loki, Promtail
- **Ingress/Gateway:** Gateway API, Envoy Gateway, MetalLB
- **Authentication:** Keycloak
- **Storage:** vSphere CSI, Longhorn
- **Backup:** Velero
- **Security:** cert-manager, Kyverno
- **GitOps:** FluxCD

### Configuration System
- Single YAML file as source of truth
- JSON Schema validation with custom business rules
- Default values per provider with override capability
- Guided configuration mode for interactive setup
- Configuration normalization and export
- Dot-notation path updates (`opencenter cluster set`)

### Operational Tools
- **Drift detection** — Compare live state against declared configuration
- **Backup/Restore** — Cluster state snapshots and recovery
- **Cluster Doctor** — Pre-deployment readiness checks and tool verification
- **Import/Scan** — Adopt existing clusters into openCenter management
- **Audit logging** — Track all CLI operations with credential masking
- **Cluster locking** — Prevent concurrent modifications

### Extensibility
- External plugin system (`opencenter-<name>` executables)
- Service plugin architecture for custom platform services
- Provider adapter interface for new infrastructure backends

## Configuration Example

```yaml
opencenter:
  cluster:
    cluster_name: production
    organization: acme-corp

  infrastructure:
    provider: openstack
    cloud:
      openstack:
        auth_url: https://identity.api.cloud.example.com/v3
        region: us-east-1

  kubernetes:
    version: 1.33.5
    control_plane_count: 3
    worker_count: 2
    cni: calico

  services:
    keycloak:
      enabled: true
    kube-prometheus-stack:
      enabled: true
    loki:
      enabled: true
    velero:
      enabled: true

secrets:
  sops:
    age_keys:
      - age1ql3z7hjy54pw3hyww5ayyfg7zqgvc7w3j2elw8zmrj2kg5sfn9aqmcac8p
```

## Quick Start

```bash
# Install tools
mise install

# Build CLI
mise run build

# Initialize a cluster
./bin/opencenter cluster init my-cluster --org my-org

# Edit configuration
$EDITOR ~/.config/opencenter/clusters/my-org/.my-cluster-config.yaml

# Validate
./bin/opencenter cluster validate my-cluster

# Generate GitOps repository
./bin/opencenter cluster generate my-cluster

# Deploy
./bin/opencenter cluster deploy my-cluster
```

**Time to first cluster:** ~10 minutes configuration + 30–50 minutes deployment

## Known Limitations (RC)

- AWS provider support is planned but not yet implemented
- ArgoCD integration is in progress (FluxCD is fully supported)
- Air-gapped deployment requires the separate openCenter-AirGap package
- Windows worker node support requires the separate opencenter-windows package

---

## Breaking Changes from Pre-Release

- Configuration file location moved to organization-based structure: `~/.config/opencenter/clusters/<org>/`
- Service configuration keys standardized across all providers
- `opencenter setup` renamed to `opencenter cluster generate`
- `opencenter bootstrap` renamed to `opencenter cluster deploy`
