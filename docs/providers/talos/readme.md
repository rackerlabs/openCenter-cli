# Talos Provider Specification Index

This directory contains the specification set AI agents will use to implement the Talos on OpenStack provider for the openCenter CLI. Start here to understand the scope of work and where to find detailed guidance.

## Documents
- [design.md](./design.md) — product intent, personas, goals, and the end-to-end experience vision.
- [architecture.md](./architecture.md) — system blueprint showing components, networks, and control flows.
- [requirements.md](./requirements.md) — prioritized product requirements, user stories, and validation criteria.

## How to Use This Spec
1. **Read `design.md`** to internalize the desired experience and constraints.
2. **Consult `architecture.md`** while implementing infrastructure templates, controllers, or integrations.
3. **Reference `requirements.md`** to track acceptance criteria, success metrics, and backlog items.
4. Update all three documents together when feature scope evolves.
